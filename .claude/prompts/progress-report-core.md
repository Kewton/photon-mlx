# 進捗レポートコアプロンプト

## 目的
プロジェクトの現在の状態を包括的に分析し、ステークホルダーが判断に使えるレポートを生成する。

## データ収集項目

### GitHub状態
- オープンIssue数と内訳（ラベル別）
- オープンPR数と状態（レビュー待ち、変更要求など）
- 最近クローズしたIssue（直近1週間）
- マイルストーンの進捗

### Git状態
- 最近のコミット履歴
- アクティブブランチ
- worktreeの状態

### コードベース状態
- ソースコード行数（`baseline_reporag/`）
- テストコード行数（`photon_mlx/tests/`）
- テスト実行結果
- ruff警告数
- 依存クレート数

## レポートフォーマット

```markdown
# 進捗レポート（YYYY-MM-DD）

## エグゼクティブサマリー
<!-- 1-2文で全体状況を要約 -->

## 数値サマリー
| 項目 | 値 |
|------|-----|
| オープンIssue | X件 |
| オープンPR | X件 |
| 今週完了 | X件 |
| ソースコード | X行 |
| テストコード | X行 |
| テストパス率 | X% |
| ruff警告 | X件 |

## 直近の成果
<!-- 完了したIssue/PRをリスト -->

## 進行中の作業
<!-- アクティブなブランチ/PRをリスト -->

## ブロッカー・リスク
<!-- 停滞しているIssue、長時間オープンのPRなど -->

## 今後の優先事項
<!-- 推奨する次のアクション -->
```

## 注意事項
- データが取得できない項目は「N/A」と記載
- 推測ではなく、実際のデータに基づいて報告
- ポジティブな面とリスクの両方をバランスよく報告
