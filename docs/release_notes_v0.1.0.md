# PHOTON-RepoRAG v0.1.0 MVP Release Notes

PHOTON-RepoRAG v0.1.0 is the first MVP release of a Multi-turn RAG system for local repositories and markdown corpora.

## Highlights

- Multi-turn RAG pipeline for follow-up questions, omitted context, and comparison questions
- Streamlit management UI for ingest, index, training, project registration, chat, and comparison mode
- Baseline vs PHOTON comparison mode with answer, citation, metrics, and retrieval debug views
- PHOTON-guided evidence selection using current-question and session-context scores
- Related past question retrieval and supplemental evidence selection for multi-turn context carryover
- Support check prompt guardrails for claims that are weakly supported by retrieved evidence
- CLI entrypoints for RAG usage and PHOTON training / generation
- Minimal baseline demo corpus under `examples/minimal_corpus/`
- Deterministic scenario scorer for `workspace/テストシナリオ2.md`
- MIT License for repository code and documentation

## Install And Start

The MVP distribution path is GitHub clone plus source checkout.

```bash
git clone https://github.com/Kewton/photon-mlx.git
cd photon-mlx

python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e ".[app]"

streamlit run app/photon_app.py --server.port 8501
```

For CLI usage after ingest / index:

```bash
photon-rag ask \
  --config configs/baseline.yaml \
  --repo-id <repo_id> \
  --question "..."
```

For a baseline-only smoke check before connecting private data:

```bash
photon-rag ingest --repo examples/minimal_corpus --repo-id minimal_demo --commit HEAD --config configs/baseline.yaml
photon-rag index --repo-id minimal_demo --config configs/baseline.yaml
photon-rag ask --repo-id minimal_demo --config configs/baseline.yaml --question "PHOTON-RepoRAG はどのような業務課題を解決しますか？"
```

## Not Included

This release does not include:

- PHOTON checkpoint weights
- External LLM weights
- Tokenizer weights
- Embedding / reranker model weights
- Local `workspace/`, `projects/`, `checkpoints/`, `.cache/`, logs, or generated local reports

Users must obtain and use external models and checkpoints under their own applicable licenses and access permissions.
